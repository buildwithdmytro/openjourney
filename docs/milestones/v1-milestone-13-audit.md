**Summary**

Milestone 13 is complete. OpenJourney now has environment-scoped feature flags with typed variants, deterministic percentage rollout via stable bucketing (reusing `experiment/assign.go`), segment/rule targeting via the audience DSL, a kill switch, exposure events, and a public IDOR-safe evaluation edge. The flag resource mirrors journeys and experiments: versioned, published via human-actor gate, with immutable versions stored as blobs. The browser SDK fetches flags, caches them with offline fallback, and emits exposures on read. The admin console section built on M12 primitives lets operators create, configure, publish, and toggle flags end-to-end. Journey nodes can branch on evaluated flags. The evaluation engine is pure, deterministic (no `math/rand`/wall-clock), and composable: targeting rules are checked first (first match wins), then rollout bucketing, then default. The public edge is rate-limited and IDOR-safe via the `byExternalID` column-pin (anonymous subjects match `anonymous_id` only, known subjects require a `SignInAppToken` and match `external_id` only). Exposure is event-sourced: the only writer of exposure state is the `ProjectEvent` projector. All 19 tasks are complete with evidence. All 621 Go tests, 273 web tests, and 30 SDK tests pass. No new dependencies added. Feature flags milestone complete.
