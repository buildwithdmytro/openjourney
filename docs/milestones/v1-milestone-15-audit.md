**Summary**

Milestone 15 is complete. OpenJourney now has reference-data catalogs and governed Connected Content (send-time external data fetching), implementing the `§5.8` content & personalization workstream. Catalogs provide parent definitions + child items (`jsonb` payload keyed by business item key) with CRUD, paginated query, and idempotent multipart bulk upload. The render-context seam (`render.RenderWithContext`) threads `ctx + principal + store + fetcher` into the Liquid engine during send-time rendering (campaigns, journeys, preview) while remaining backward-compatible with bare `render.Render`. Personalization filters (`catalog_item`) perform store-backed, cache-first lookups with graceful fallback on missing items. Governed Connected Content sources enforce host allowlisting, `*_ref`-only authentication, per-source timeouts, circuit breakers, human-actor publication gating, secret redaction on read, and RBAC scope enforcement (`catalogs:read`/`catalogs:write`). The `connected_content` Liquid tag performs SSRF-guarded fetching (validating dial-time IP safety via `channels.IsSafeURL`/`IsPrivateIP`), caches responses in an in-process bounded TTL cache (`TTLCache`), emits activity audit logs, and degrades gracefully to fallbacks on any error without failing message delivery. Admin UI (`Catalogs.tsx`) provides tabbed management using M12 primitives. End-to-end integration and security test suites verify all personalized delivery and SSRF lockdown contracts. All 18 sub-tasks (20.0.1–20.9.4) are complete with verifying tests. All 628 Go tests, 310 web tests, and 30 SDK tests pass. Zero new dependencies added. Catalogs & Connected Content milestone complete.
